## Overview

This document defines the public interface contracts for the Essentiel web application. These contracts describe how users and external systems interact with the web app.

---

## Web Routes

The web app is a **single-page application (SPA)** with hash-based routing.

### Base URL

```text
Production: https://lemra-org.github.io/essentiel_app/
Development: http://localhost:<port>/
```

### Route Table

| Route | Description | State |
|-------|-------------|-------|
| `/#/` | Main game screen (default) | Card display, shuffle, filters |
| `/#/about` | About screen | App info, version, credits |

**Routing Strategy**: Hash-based (`#/`) to support GitHub Pages static hosting without server-side configuration.

**Deep Linking**: All routes support direct URL access and browser back/forward navigation.

## Browser Local Storage

The web app uses browser `localStorage` (via Flutter's `shared_preferences` package) to persist user preferences.

### Storage Keys

| Key | Type | Description | Example Value |
|-----|------|-------------|---------------|
| `flutter.CATEGORY_FILTER_PREF_KEY` | JSON Array | Selected category filters | `["Famille","Parent - Enfant"]` |
| `flutter.shared_preferences_first_launch` | Boolean | First launch flag | `false` |

**Key Prefix**: All keys prefixed with `flutter.` by `shared_preferences` package to prevent collisions.

**Storage Limit**: 5MB per origin (lemra-org.github.io)

**Persistence**:
- Data survives browser restarts
- Data lost on browser cache clear
- Each origin (domain:port) has separate storage

### Data Format

**Category Filter Preference**:
```json
{
  "flutter.CATEGORY_FILTER_PREF_KEY": "[\"Famille\",\"Parent - Enfant\",\"Couple\"]"
}
```

**Validation**:
- Categories must exist in loaded category list
- Invalid categories silently filtered on load
- Empty array treated as "all categories selected"

## Service Worker API

The web app uses a service worker for offline functionality and caching.

### Service Worker Location

```text
/flutter_service_worker.js (auto-generated by Flutter)
```

### Caching Strategy

- **Static assets** (HTML, CSS, JS, images): Cache-first
- **Dynamic data** (Google Sheets API): Network-first with fallback to cache
- **Cache name**: Versioned per deployment (`flutter-cache-v{VERSION}`)

### Offline Behavior

| Resource | Online | Offline (cached) | Offline (not cached) |
|----------|--------|------------------|----------------------|
| App shell | Network-first | Serve from cache | Error page |
| Card data | Network-first | Serve from cache | "No connection" message |
| Images | Cache-first | Serve from cache | Placeholder |

### Update Mechanism

- Service worker checks for updates every 30 minutes
- User notified of available updates via banner
- User can manually trigger update via refresh menu item
- Old caches cleared on update

## External API Dependencies

### Backend API (Essentiel Data Service)

**Base URL**: Configured via environment (to be provided by team)

Example environments:
- Development: `http://localhost:3000` (local backend for testing)
- Production: `https://api.essentiel.example.com` (to be confirmed)

**Authentication**: None required (public read-only endpoints)

**Endpoints**:

#### GET /api/categories

Fetch all question categories.

**Request**:
```http
GET /api/categories HTTP/1.1
Host: api.essentiel.example.com
```

**Response** (200 OK):
```json
{
  "categories": [
    {
      "name": "Famille",
      "color": "#FF9800"
    },
    {
      "name": "Parent - Enfant",
      "color": "#9C27B0"
    },
    {
      "name": "Couple",
      "color": "#E91E63"
    }
  ]
}
```

**Fields**:
- `name` (string, required): Category display name
- `color` (string, required): Hex color code with `#` prefix

#### GET /api/questions

Fetch all question cards.

**Request**:
```http
GET /api/questions HTTP/1.1
Host: api.essentiel.example.com
```

**Response** (200 OK):
```json
{
  "questions": [
    {
      "question": "Quelle est ta plus grande fierté cette année?",
      "category": "Famille",
      "forCouples": false,
      "forFamilies": true,
      "forParentChild": false
    },
    {
      "question": "Qu'est-ce qui te fait te sentir aimé(e)?",
      "category": "Couple",
      "forCouples": true,
      "forFamilies": false,
      "forParentChild": false
    }
  ]
}
```

**Fields**:
- `question` (string, required): Question text
- `category` (string, required): Category name (must match a category from `/api/categories`)
- `forCouples` (boolean, required): Card suitable for couples
- `forFamilies` (boolean, required): Card suitable for families  
- `forParentChild` (boolean, required): Card for parent-child context

**Note**: `isForInternalMood` is derived client-side by checking if question contains "météo" (lowercase)

**CORS Headers**: Backend must include appropriate CORS headers for lemra-org.github.io:

```http
Access-Control-Allow-Origin: https://lemra-org.github.io
Access-Control-Allow-Methods: GET, OPTIONS
Access-Control-Allow-Headers: Content-Type
```

**Rate Limits**: Backend-defined (recommend 100 requests/minute per IP)

**Caching Headers**: Recommended to reduce backend load:

```http
Cache-Control: public, max-age=300, stale-while-revalidate=600
```

This allows browsers to cache responses for 5 minutes, reducing API calls.

**Error Handling**:

| Status Code | Meaning | Client Action |
|-------------|---------|---------------|
| 200 OK | Success | Parse and use data |
| 404 Not Found | Endpoint not found | Show error, use cached data if available |
| 429 Too Many Requests | Rate limit exceeded | Retry after delay, use cached data |
| 500 Internal Server Error | Backend error | Show error, use cached data if available |
| 503 Service Unavailable | Backend down | Show "service unavailable" message, use cache |
| Network error | No connection | Show "no connection" message, use cache |

**Fallback Strategy**: Web app should cache API responses in localStorage and use cached data when API is unreachable.

## Security Considerations

### Content Security Policy (CSP)

Recommended CSP headers for GitHub Pages (set via meta tag in `index.html`):

```html
<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; 
               script-src 'self' 'unsafe-inline' 'unsafe-eval'; 
               style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; 
               font-src 'self' https://fonts.gstatic.com; 
               img-src 'self' data:; 
               connect-src 'self' https://sheets.googleapis.com;">
```

**Note**: Flutter web requires `unsafe-inline` and `unsafe-eval` for dart2js compiled code. WebAssembly builds may tighten these restrictions.

### Credential Exposure

**Web builds** (zero credentials):
- No Google Sheets credentials embedded
- No API keys embedded
- Only backend API URL (public information)
- Backend handles all authentication server-side
- Clean security boundary between client and data source

**Mobile builds**:
- Service Account credentials embedded in compiled binaries (not readable by users)
- Follows Google's recommended practices
- Can optionally migrate to backend API for consistency

## Accessibility Contracts

### Keyboard Navigation

| Action | Keyboard Shortcut |
|--------|------------------|
| Shuffle cards | `Space` or `Enter` (when shuffle button focused) |
| Open menu | `Tab` to menu button, `Enter` to activate |
| Navigate menu items | `Arrow Up/Down` |
| Close menu | `Escape` |
| Flip card | `Enter` or `Space` (when card focused) |

### Screen Reader Support

- Semantic HTML structure (headings, landmarks)
- ARIA labels for icon-only buttons
- Focus indicators for all interactive elements
- Card content announced on flip

### Touch Target Sizes

- Minimum: 48x48 CSS pixels (WCAG 2.1 Level AAA)
- Menu items: 56px height (Material Design spec)
- Card flip area: Full card height (30% of viewport)

```html
<script>
```

### Breaking Changes Policy

- **URL structure**: No changes to route paths without migration notice
- **localStorage keys**: No changes to key names without migration code
- **PWA manifest**: Changes require user reinstallation notice
- **Service worker**: Auto-updates on next app launch
